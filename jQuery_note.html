JavaScript: The Definitive Guide

The jQuery Function

Four ways to invoke $() function.

  1. to select DOM elements: $(selector, context) (NOTE: it returns jQuery objects)
  2. to convert raw DOM elements to jQuery objects: $(this), $(document)... (NOTE: passing an array of elements to $() are accepted)
  3. to create new elements, for example: var img = $('<img/>', {src:url, css: {borderWidth:5}, click:handleClick})
  4. the jQuery version of the onLoad() function, $(f) has a moe verbose form $(document).ready(f)

jQuery.noConflict()

jQuery.noConflict(); // Restore $ to its original state
jQuery(function($) { // Use $ as a local alias
// Put all your jQuery code here
});

jQuery objects ⇋ raw DOM elements

  1. .get() and [i] convert a single one
  2. .toArray() convert all of them to a true array
  3. see the second way to invoke $() function

jQuery object property

1. .length .size() method can return this value
2. [i] numeric properties from 0 to length -1. NOTE: doing this will convert the jQuery object to a raw DOM element!
3. .selector the selector string (if any) that was used when the jQuery object was created
4. .context the context object that was passed as the second argument to $()
5. jquery the jQuery version number as a string. Testing for the existence of this property is a simple way to distinguish jQuery objects from other array-like objects.

jQuery selector

id and class selector

  • #id if there are more than one, only return the first
  • .class

attribute selector

  • [attr] have an attr attribute (regardless of its value)
  • [attr=val] attr attribute whose value is val
  • [attr!=val] have no attr attribute, or is not equal to val (jQuery extension)
  • [attr^=val] begins with val
  • [attr$=val] ends with val
  • [attr*=val] contains val
  • [attr~=val] Matches elements whose attr attribute, when interpreted as a list of words separated by spaces, includes the word val . Thus the selector “div.note” is the same as “div[class~=note]”.
  • [attr|=val] Matches elements whose attr attribute has a value that begins with val and is optionally followed by a hyphen and any other characters.

form type selector

  • :button
  • :checkbox
  • :file
  • :image
  • :input Matches user input elements: <input> , <textarea>, <select> , and <button> (jQuery extension).
  • :password
  • :radio
  • :reset
  • :submit
  • :text

status selector

  • :disabled
  • :enabled
  • :hidden
  • :selected Matches <option> elements that are selected. Use :checked for selected checkboxes and radio buttons (jQuery extension).
  • :visible

position selector

  • :eq(n) and :nth(n)
  • :lt(n) and :gt(n)
  • :first and :last
  • :odd and :even

DOM tree selector

  • :first-child
  • :last-child
  • :nth-child(n) n can be a number, the word “even”, the word “odd”, or a formula.
  • :only-child Matches elements that are the only child of their parent.
  • :parent Matches elements that are parents. This is the opposite of :empty (jQuery extension).
  • :empty Matches elements that have no children, including no text content.
  • :has(sel) Matches elements that have a descendant matching the nested selector sel .
  • :not(sel) Matches elements that are not matched by the nested selector sel

other selector

  • :contains(text) Finding elements based on textual content.
  • :header Matches any header element: <h1>, <h2>, <h3>, <h4>, <h5>, or <h6> (jQuery extension).
  • :animated Matches elements that are currently being animated by jQuery.

Selection Methods

many of the methods provide the same functionality as the selector grammar itself. Note that these methods return a jQuery object with a single element. This is different than regular array indexing, which returns a single element with no jQuery object wrapped around it

  • .first() and .last()
  • .eq()
  • .slice() works like the Array.slice() method
  • .filter() argument can be a selector or a predicate function. See also jQuery.grep()
  • .not() just like filter(), except that it inverts the sense of the filter
  • .add()augments a selection rather than filtering or refining it, also removes duplicate elements and sorts the combined selection so that the elements are in document order
  • .find()
  • .children() returns the immediate child elements of each selected element, filtering them with an optional selector.
  • .contents() similar to children(), but it returns all child nodes, including text nodes, of each element.Also, if any of the selected elements is an <iframe>, contents() returns the document object for the content of that <iframe>. Note that contents() does not accept an optional selector string
  • .next() and .prev()
  • .nextAll() and .prevAll()
  • .nextUntil() and .prevUntil()
  • .contains()
  • .is()
  • .has()
  • .siblings()
  • .parent() returns the parent of each selected element
  • .parentUntil() returns the ancestors of each selected element until the first ancestor that matches the specified selector.
  • .closest() For this method, an element is considered an ancestor of itself. In jQuery 1.4, you can also pass an ancestor element as the second argument to closest(), to prevent jQuery from climbing the ancestor tree beyond the specified element:

examples:
$("a[href]").closest("div") // Innermost <div>s that contain links
$("a[href]").parentsUntil(":not(div)") // All <div> wrappers directly around <a>

Reverting to a previous selection

  • .end() Calling end() in a method chain restores the set of matched elements to its previous state.
  • .pushStack()
  • .andSelf() returns a new jQuery object that includes all of the elements of the current selection plus all of the elements (minus duplicates) of the previous selection. andSelf() works like the add() method, and “addPrev” might be a more descriptive name for it.

    • examples:
    var sel = $("div"); // Select all <div> elements
    sel.pushStack(document.getElementsByTagName("p")); // Modify it to all <p> elts
    sel.end(); // Restore <div> elements

These jQuery methods are like the classList methods

4. Getting and Setting HTML Form Values

  • val()

5. Getting and Setting Element Content

  • text() works even in browsers that do not support the textContent or innerText properties
  • html() jQuery uses the innerHTML property to do this: x.html() is effectively the same as x[0].innerHTML

6. Getting and Setting Element Geometry

  • offset() This method measures positions relative to the document and returns them in the form of an object with left and top properties that hold the X and Y coordinates. If you pass an object with these properties to the method, it sets the position you specify. It sets the CSS position attribute as necessary to make elements positionable
  • position() similar to offset(), except that it is a getter only and it returns element positions relative to their offset parent, rather than relative to the document as a whole.
  • offsetParent() maps each element to the nearest positioned ancestor element or to the <body> element.
  • width() and height() return the basic width and height and do NOT include padding, borders, or margins
  • innerWidth() and innerHeight() return the width and height of an element plus the width and height of its padding
  • outerWidth() and outerHeight() normally return the element’s dimensions plus its padding and border. If you pass the value true to either of these methods, they also add in the size of the element’s margins
  • scrollTop() and scrollLeft() query the scrollbar positions for an element or set the scrollbar positions for all elements. Unlike other setters, they can NOT take function as argument

7. Getting and Setting Element Data

  • data() jQuery also defines utility function forms of the data() and removeData() methods
  • removeData()

Altering Document Structure

1. Inserting and Replacing Elements

operation $(target).method(content) $(content).method(target)
insert content at end of target append() appendTo()
insert content at start of target prepend() prependTo()
insert content after target after() insertAfter()
insert content before target before() insertBefore()
replace target with content replaceWith() replaceAll()

2. Copying Elements

clone() does not normally copy event handlers or other data you have associated with elements; pass true if you want to clone that additional data as well.

3. Wrapping Elements

  • wrap() wraps each of the selected elements
  • wrapInner() wraps the contents of each selected element
  • wrapAll() wraps the selected elements as a group

4. Deleting Elements

  • empty()
  • remove() Note that it is not necessary to remove elements before reinserting them into the document: you can simply insert them at a new location and they will be moved.
  • detach() works just like remove() but does not remove event handlers and data. detach() may be more useful when you want to temporarily remove elements from the document for later reinsertion
  • unwrap() removes the parent element of each selected element. It does NOT accept an optional selector argument
